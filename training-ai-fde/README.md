## How this repo works

- **One folder per week** — `week-00/`, `week-01/`, … `week-12/`.
- **This README is the index** — every week below links to the official lesson and to my own work folder.
- **Submit = push the work, then share the link** — a file, a folder, a pull request, or a release tag, as each week specifies.
- **Public on purpose** — a public repo forces clean commits, readable READMEs, and honest history from day one.
- **No secrets in git** — API keys live in environment variables, never in a committed file.


## Program at a glance

| Phase | Weeks | Focus |
|---|---|---|
| **Phase 0 — Setup** | Week 0 | Tooling, accounts, a public submission repo, and a calibration diagnostic |
| **Phase 1 — Production Engineering Foundations** | Weeks 1–3 | Unlearning coursework habits, learning how code ships and survives |
| **Phase 2 — AI Application Engineering** | Weeks 4–7 | Building reliable systems on top of unreliable models |
| **Phase 3 — The FDE Craft** | Weeks 8–10 | Scoping, integration, and customer communication |
| **Phase 4 — Capstone Embedded Deployment** | Weeks 11–12 | A real deployment with a real owner and a real deadline |
